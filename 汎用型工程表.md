# 汎用型工程表 — ローカルAI秘書システム

> このファイルは AINAS プロジェクトの工程表を汎用化したテンプレートです。
> 個人名・エージェント名・LLM名をプレースホルダに置換済み。
> **Local First / No 3D・AR** を既定とします。
> 進行は1タスクずつ。「次へ」で前進。日本語で出力。

---

## プレースホルダ一覧（使用前に置換してください）

| プレースホルダ | 内容 | 例 |
|---|---|---|
| `<USER_NAME>` | ユーザーの呼称 | 松村、田中 |
| `<AGENT_NAME>` | AIエージェントの名前 | AINAS、ARIA |
| `<LLM_NAME>` | 使用するローカルLLM | llama3、gemma2 |
| `<VAULT_PATH>` | Obsidian Vault のパス | ~/my-vault |
| `<PROJECT_DIR>` | プロジェクトルートパス | ~/my-agent |

---

## 設計方針（固定）

- **Local First**：稼働時LLMはOllama限定。RAGはObsidian限定。外部API禁止（補助保存用途のGoogle Docs/Sheetsのみ例外）
- **No 3D/AR/VRM/Unity**：キャラ表示は2D動画のみ
- **承認制**：ファイル書き込み・削除・公開はユーザー承認後のみ
- **1タスクずつ**：R-8（6項目事前確認）→実装→承認→次へ

## PHASE 1 — 環境構築

- **目的**: 開発・稼働の土台を作る
- **成功条件**:
  - [ ] 1-1 既存の不要要素を全除去
  - [ ] 1-2 Python 3.11+ と仮想環境を構築
  - [ ] 1-3 Ollama 導入・`<LLM_NAME>` モデル取得・オフライン動作確認
  - [ ] 1-4 Obsidian 導入・`<VAULT_PATH>` 確定・`raw/ wiki/ Memory/ Knowledge/ Daily/` 作成
  - [ ] 1-5 ベクトルDB（Chroma 推奨）ローカル起動確認
  - [ ] 1-6 FastAPI 雛形 + ヘルスチェックエンドポイント
  - [ ] 1-7 WebSocket チャネル開設
  - [ ] 1-8 状態管理モジュール（インメモリ＋ローカル JSON 永続化）
  - [ ] 1-9 ディレクトリ構成作成（`3d/ ar/ vrm/ unity/` を作らない）
  - [ ] 1-10 React + Vite フロントエンド初期化（localhost 通信のみ）
- **禁止**: 3D/AR/VRM/Unity モジュール作成 / Vault への機密保存

---

## PHASE 2 — LLM 接続

- **目的**: 稼働時LLMをOllamaに固定して接続する
- **成功条件**:
  - [ ] 2-1 FastAPI → Ollama（ローカル）接続
  - [ ] 2-2 システムプロンプト実装（`<AGENT_NAME>` の人格定義）
  - [ ] 2-3 ネット切断状態で応答が返る
- **禁止**: クラウドLLM呼び出し / エージェントによる最終決定

---

## PHASE 3 — Obsidian RAG

- **目的**: Obsidianを情報源とするローカルRAGを構築する
- **成功条件**:
  - [ ] 3-1 Vault 読み取り専用アクセス（ローカルMarkdownのみ）
  - [ ] 3-2 チャンク化 + ローカル埋め込み → ベクトルDB格納
  - [ ] 3-3 RAG 検索（関連ノート取得 → Ollama へ供給）
  - [ ] 3-4 オフラインでRAG検索が動作
- **禁止**: Web 取得 / スクレイピング / クラウド埋め込み

---

## PHASE 4 — 会話機能

- **目的**: RAG・記憶・自動成長ナレッジを含む会話を成立させる
- **成功条件**:
  - [ ] 4-1 基本チャットパス（入力→RAG→Ollama→応答）
  - [ ] 4-2 「覚えて」記憶書込（提案→承認→Memory/ へ書込）
  - [ ] 4-3 `raw/` 投入受付（ローカルファイルのみ）
  - [ ] 4-4 一発整理パス（ローカルOllama・提案→承認→`wiki/` へ相互リンク出力）
  - [ ] 4-5 ナレッジグラフ自動マッピング（提案のみ）
  - [ ] 4-6 会話同期（会話記録をVaultに蓄積・起動時参照）
  - [ ] 4-7 再利用一言運用（「wiki/ のナレッジベースを使って」で全体参照）
- **禁止**: 無承認書込 / クラウド送信

---

## PHASE 5 — 音声認識（STT）

- **目的**: 音声入力をテキスト化する
- **成功条件**:
  - [ ] 5-1 Whisper（faster-whisper 推奨）でSTT実装
  - [ ] 5-2 STT テキストを会話パイプラインへ接続
  - [ ] 5-3 オフラインでSTT動作
- **禁止**: クラウドSTT利用

---

## PHASE 6 — 音声合成（TTS）

- **目的**: 応答を音声出力する
- **成功条件**:
  - [ ] 6-1 ローカルTTS実装（macOS: `say` コマンド / Linux: espeak-ng 等）
  - [ ] 6-2 TTSを会話パイプラインへ接続・再生中インジケーター
  - [ ] 6-3 オフラインでTTS動作
- **禁止**: クラウドTTS / 高音・早口・過剰演技

---

## PHASE 7 — 顔認証

- **目的**: 本人のみが機能を利用できる状態にする
- **前提確認**: カメラ使用可否・登録名方式をユーザーに確認してから実装開始
- **成功条件**:
  - [ ] 7-1 初回登録フロー（顔取得→embedding保存→氏名→呼称→プロファイル生成）
  - [ ] 7-2 通常起動フロー（認証→セッション→機能解放→終了処理）
  - [ ] 7-3 認証失敗時ルール（機能をすべてロック）
  - [ ] 7-4 複数人ルール（登録者以外を拒否）
  - [ ] 7-5 名前確認（登録名一致で全機能解放）
  - [ ] 7-6 セッション管理（アイドルタイムアウト・自動ログアウト）
  - [ ] 7-7 認証ボタン（開始/終了）実装
- **禁止**: クラウド認証 / プロファイルのクラウド同期 / 常時カメラ起動

---

## PHASE 8 — 離席検知

- **目的**: 離席・着席・来客を管理する
- **成功条件**:
  - [ ] 8-1 インターバル監視（10秒無操作→1〜2秒スナップ→判定→即停止・画像非保存）
  - [ ] 8-2 離席ボタン（離席状態→監視継続→来客フラグ管理開始）
  - [ ] 8-3 離席中の他者検知（固定文発話・顔非保存・カウントのみ）
  - [ ] 8-4 着席ボタン（カメラ＋本人認証→来客有無を報告）
- **禁止**: 常時監視 / 画像保存 / 個人名発話 / 他者顔保存

---

## PHASE 9 — 感情分析

- **目的**: 状態に応じた提案メッセージを出す（強制なし）
- **成功条件**:
  - [ ] 9-1 感情分析実装（提案のみ・強制動作なし）
  - [ ] 9-2 状態連動メッセージ（疲労・眠気・通常 の3分岐）
- **禁止**: 強制動作 / 画像保存 / 感情を理由とした自律動作

---

## PHASE 10 — 会議システム

- **目的**: 外部会議システムに秘書として参加し、ログを補助保存する
- **前提確認**: 会議システムのURL・Google認証情報をユーザーに確認してから実装
- **成功条件**:
  - [ ] 10-1 外部会議システムへ参加者として接続
  - [ ] 10-2 参加構成で秘書人格維持
  - [ ] 10-3 会議での役割（提案/分析/整理・決定はユーザーのみ）
  - [ ] 10-4 議事録保存（補助保存のみ。日時・提案・決定・未解決を含む）
  - [ ] 10-5 会議I/O（音声入出力・チャットログ・会議ログ）
- **禁止**: 補助保存以外のGoogle利用 / エージェントによる決定

## PHASE 11 — 動画機能（2D動画のみ）

- **目的**: キャラクターを2D動画で表示し、黒画面なく遷移させる
- **前提確認**: 動画ファイル（待機/話す/アイドリング等）の用意をユーザーに確認
- **成功条件**:
  - [ ] 11-1 `待機` 既定アイドル表示
  - [ ] 11-2 `話す`（音声出力中ループ→終了でクロスフェードで`待機`へ）
  - [ ] 11-3 `アイドリング`（N秒無操作でランダム→`待機`へクロスフェード）
  - [ ] 11-4 全遷移にクロスフェード適用（黒画面ゼロ）
  - [ ] 11-5 Reactに動画ステージ実装（2Dのみ）
- **禁止**: 黒画面遷移 / ハードカット / 3D・AR・VRM

- **目的**: 外出シーケンス→ローカル検索→帰宅シーケンスを成立させる
- **前提確認**: 外出/帰宅動画ファイルの用意をユーザーに確認
