## システム定義

- 名称（コードネーム）：**AINAS**
- 一人称：**「アイナス」**
- 種別：**個人専用ローカルAI秘書**（個人1名のみが利用する）
- 形態：localhost で動作するローカルアプリ（オフライン動作可能）
- 非該当：顧客対応チャットボットではない（顧客対応AIとは完全分離：`00_絶対ルール.md` R-5）

-----

## 目的（実装対象スコープ）

AINAS は以下の機能群を提供する。詳細仕様は `05_機能仕様.md`、実装順は `04_工程表.md`。

1. 個人ナレッジ管理（Obsidian Vault）
1. 開発支援
1. ライフログ / 状態メモリ
1. 音声メモ
1. アイデア整理
1. 作業アシスト
1. 会議アシスト（GAS会議システム参加）
1. キャラクターAI（2D動画表示）

-----

## 中核コンセプト：自動成長する「第2の脳」

- 【目的】Obsidian Vault を、セッションをまたいで消えない長期記憶として育てる
- 【入力】`raw/` に投入されたローカルファイル（メモ・テキスト・PDF・保存済みWeb記事ファイル）
- 【出力】`wiki/` に出力された構造化Markdown（相互リンク付き）＋自動形成されるナレッジグラフ
- 【成功条件】1度の整理パスで `raw/` の内容が `wiki/` に構造化・相互リンクされ、Obsidianのグラフビューに反映される
- 【禁止事項】Web自動取得（`00_絶対ルール.md` R-7）/ クラウド送信（R-4）
- 【スローガン】“Your brain forgets. AINAS doesn’t.”

3ステップ運用：

1. `raw/` に放り込む
1. ローカルOllamaが1度だけ整理・構造化・相互リンク
1. `wiki/`（Obsidian）に綺麗なMarkdownとして出力

## システム構成（全体像）

- 中央リレー：FastAPI（localhost）
- リアルタイム通知：WebSocket（localhost）
- LLM：Ollama（ローカル・稼働時唯一のLLM）
- 知識ベース / RAG：Obsidian Vault + ローカルベクトルDB（Qdrant または Chroma）
- 音声入力：Whisper.cpp（ローカルSTT）
- 音声出力：ローカルTTS
- 顔認証：DeepFace / face_recognition（ローカル）
- 感情分析：FER（ローカル・提案のみ）
- フロントエンド：React（localhost API / WebSocket のみ通信）
- ブラウザ補助：Chrome拡張（キャラ表示補助のみ）
- 会議ログ補助保存：GAS + Google Docs / Spreadsheet（補助保存用途のみ：`00_絶対ルール.md` R-4b）

詳細な採用技術・バージョン・ライブラリは `03_技術要件.md`。

## 利用フロー（概要）

1. 起動 → 「認証開始」ボタン表示
1. 顔認証成功 → セッション発行 → プロファイル読込 → 機能解放 → 「終了」ボタン表示
1. 利用中 → チャット / RAG / 音声 / 離席検知 / 感情分析 / 会議 / 動画表示
1. 「終了」 → セッション破棄 → キャッシュ・一時メモリ消去 → AI停止

詳細は `05_機能仕様.md`。
