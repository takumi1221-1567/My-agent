## RAG（PHASE3）

### F-RAG01 Vault読み取り

- 【目的】ローカルVaultを情報源として読む
- 【入力】Vault内Markdownファイル
- 【出力】読み取り済みテキスト
- 【成功条件】Vault内Markdownのみを読み取り、外部を一切参照していない
- 【禁止事項】書き込み / Web取得 / スクレイピング

### F-RAG02 埋め込み・格納

- 【目的】検索可能な状態にする
- 【入力】Vaultテキスト
- 【出力】ベクトルDB（Qdrant/Chroma）内の埋め込み
- 【成功条件】ローカル埋め込みモデルでベクトル化し、ローカルDBに格納完了
- 【禁止事項】クラウド埋め込みAPI利用

### F-RAG03 RAG検索

- 【目的】関連ノートを応答に反映する
- 【入力】ユーザー入力
- 【出力】関連ノート上位N件をOllamaへ供給
- 【成功条件】入力に対し関連ノートが取得され、応答に反映されている
- 【禁止事項】Web検索結果の混入

## 会話・記憶・自動成長ナレッジ（PHASE4）

### F-CHAT01 基本チャット

- 【目的】RAG付き会話を成立させる
- 【入力】ユーザー入力
- 【出力】敬語応答
- 【成功条件】入力→RAG→Ollama→敬語応答 の経路で応答が返る
- 【禁止事項】タメ口 / ローカルAIによる最終決定

### F-MEM01 「覚えて」記憶書込

- 【目的】ユーザー承認の上で記憶を保存する
- 【入力】「覚えて」を含む入力
- 【出力】`Memory/` 内のMarkdown記憶ノート
- 【成功条件】保存内容を提案→ユーザー承認後にのみ書き込んでいる
- 【禁止事項】無承認の自動書込

### F-KB01 raw投入受付

- 【目的】整理対象を受け付ける
- 【入力】`raw/` に置かれたローカルファイル（メモ/テキスト/PDF/保存済みWeb記事ファイル）
- 【出力】整理待ちファイル一覧
- 【成功条件】ローカルファイルのみを対象にしている
- 【禁止事項】Web自動取得

### F-KB02 一発整理パス（raw→wiki）

- 【目的】1度の処理で構造化・相互リンクする
- 【入力】`raw/` の内容
- 【出力】`wiki/` 内の構造化Markdown（相互リンク付き）
- 【成功条件】ローカルOllamaで整理し、提案→承認後に `wiki/` へ相互リンク付き出力している
- 【禁止事項】無承認書込 / クラウド送信

### F-KB03 ナレッジグラフ自動マッピング

- 【目的】プロジェクト間の繋がりを可視化する
- 【入力】1プロンプト
- 【出力】ノート間リンク提案（Obsidianグラフに反映）
- 【成功条件】未接続ノート間の関連を提案として提示している（編集は承認後）
- 【禁止事項】無承認の自動編集

### F-KB04 会話同期

- 【目的】セッションをまたいで文脈を保持する
- 【入力】ローカルAIの会話記録
- 【出力】Vaultに蓄積された会話ログ
- 【成功条件】会話記録がVaultに蓄積され、起動時に過去文脈を参照している
- 【禁止事項】クラウド保存

### F-KB05 再利用一言運用

- 【目的】再セットアップ不要で全体参照する
- 【入力】「wiki/ のナレッジベースを使って」
- 【出力】wiki全体を参照した応答
- 【成功条件】当該一言で全ナレッジベースが参照対象になる
- 【禁止事項】毎回の全ファイル再取り込み

## 音声（PHASE5 / PHASE6）

### F-STT01 音声認識

- 【目的】音声をテキスト化する
- 【入力】マイク音声
- 【出力】Whisper.cpp によるテキスト
- 【成功条件】オフラインで音声がテキスト化される
- 【禁止事項】クラウドSTT

### F-TTS01 音声合成

- 【目的】応答を音声出力する
- 【入力】応答テキスト
- 【出力】低音・落ち着き・明瞭・中速の音声
- 【成功条件】オフラインで音声出力され、音声人格基準を満たす
- 【禁止事項】クラウドTTS / 高音・早口・過剰演技

## 顔認証（PHASE7）

### F-AUTH01 初回登録

- 【目的】本人プロファイルを作る
- 【入力】顔画像 / 氏名 / 希望呼称
- 【出力】local_profile（`fullname`/`preferred_name`/`face_embedding`/`local_settings`）
- 【成功条件】顔embedding・氏名・呼称をローカル保存し、4項目のみのprofileを生成
- 【禁止事項】4項目以外の保持 / クラウド同期

### F-AUTH02 通常起動フロー

- 【目的】本人認証で機能解放する
- 【入力】「認証開始」操作・顔画像
- 【出力】セッション発行・プロファイル読込・機能解放・「終了」ボタン
- 【成功条件】認証成功でセッション発行と機能解放、終了でセッション破棄・キャッシュ消去
- 【禁止事項】認証前の機能解放

### F-AUTH03 認証失敗時ルール

- 【目的】失敗時の機能を封じる
- 【入力】認証失敗
- 【出力】機能ロック状態
- 【成功条件】AI解放/Vault/会議/Chrome連携がすべて禁止される
- 【禁止事項】失敗時の部分解放

### F-AUTH04 複数人ルール

- 【目的】登録者以外を排除する
- 【入力】複数人または非登録者
- 【出力】拒否
- 【成功条件】登録者以外にセッション・AI解放を与えない
- 【禁止事項】他者への解放

### F-AUTH05 名前確認

- 【目的】名前一致で全機能解放を判定する
- 【入力】認識後の名前
- 【出力】一致→全機能解放／不一致→固定文発話（`02_人格設定.md` 名前開示ポリシー）
- 【成功条件】登録名一致時のみ全機能解放し、不一致時は固定文のみ発話
- 【禁止事項】不一致時の機能解放 / 個人名発話

### F-AUTH06 セッション管理

- 【目的】放置時の安全を保つ
- 【入力】操作なしの経過時間
- 【出力】自動ログアウト・セッション破棄・キャッシュ消去
- 【成功条件】アイドルタイムアウトで自動ログアウトし一時メモリを消去
- 【禁止事項】無期限セッション維持

## 離席検知（PHASE8）

### F-PRES01 インターバル監視

- 【目的】常時監視せず短時間だけ確認する
- 【入力】操作なし状態
- 【出力】10秒無操作→カメラ1〜2秒スナップ→判定→即停止
- 【成功条件】10秒無操作で起動、撮影1〜2秒、画像を保存せず判定後に即停止
- 【禁止事項】常時監視 / 画像保存

### F-PRES02 離席ボタン

- 【目的】離席状態を開始する
- 【入力】離席ボタン操作
- 【出力】離席状態 / 監視継続 / 来客フラグ管理開始
- 【成功条件】押下で離席状態へ遷移し来客フラグ管理が始まる
- 【禁止事項】離席中の機能フル解放

### F-PRES03 離席中の他者検知

- 【目的】来客対応と個人名保護を両立する
- 【入力】離席中の他者検知
- 【出力】固定文発話（`02_人格設定.md`）
- 【成功条件】個人名を発話せず固定文のみ発話し、顔を保存せずカウントのみ
- 【禁止事項】個人名発話 / アドリブ / 伝言受取 / 顔保存

### F-PRES04 着席ボタン

- 【目的】復帰時に来客を報告する
- 【入力】着席ボタン操作
- 【出力】カメラ＋本人認証→来客有無の報告
- 【成功条件】認証後に以下を発話：

```
お戻りなさいませ。先ほどお客様がいらしておりました。   （来客あり）
お戻りなさいませ。ご不在中のお客様はございませんでした。 （来客なし）
```

- 【禁止事項】認証なしの復帰

## 感情分析（PHASE9）

### F-EMO01 FER感情分析

- 【目的】状態を推定する（提案のみ）
- 【入力】インターバル撮影スナップショット
- 【出力】状態判定（疲労/眠気/通常 等）
- 【成功条件】判定結果を提案としてのみ用い、強制動作をしない
- 【禁止事項】強制動作 / 画像保存

### F-EMO02 状態連動メッセージ

- 【目的】状態に応じた声かけをする
- 【入力】F-EMO01の状態判定
- 【出力】以下の固定メッセージ：

```
お疲れではないですか？少し休憩されてはいかがでしょう。   （疲労/悲しみ）
少しお眠そうでしょうか。お飲み物はいかがですか？         （眠気）
引き続きよろしくお願いいたします。                       （集中/通常）
```

- 【成功条件】状態に応じて上記いずれかを提案として発話
- 【禁止事項】感情を理由とした自律動作

## 会議システム（PHASE10）

### F-MTG01 GAS接続

- 【目的】会議に参加者として接続する
- 【入力】GAS会議システム
- 【出力】参加状態
- 【成功条件】参加者としてGAS会議に接続できる
- 【禁止事項】GASによる意思決定

### F-MTG02 参加構成・人格維持

- 【目的】秘書として会議に在席する
- 【入力】会議（ユーザー＋ローカルAI＋顧客対応AI）
- 【出力】秘書人格での発話
- 【成功条件】会議中、敬語・秘書人格を維持している
- 【禁止事項】顧客対応AIとのデータ共有（R-5）

### F-MTG03 会議での役割

- 【目的】提案役に徹する
- 【入力】会議の議題・状況
- 【出力】提案/分析/過去知識参照/類似事例/整理/改善提案
- 【成功条件】最終決定をユーザーに委ねている
- 【禁止事項】ローカルAIによる決定

### F-MTG04 議事録保存（補助保存）

- 【目的】議事録を補助保存する
- 【入力】会議内容
- 【出力】Google Docs保存→Drive URL取得→Spreadsheet記録
- 【成功条件】議事録に「日時・提案・決定・未解決」を含み、補助保存用途に限定されている
- 【禁止事項】GoogleサービスのLLM/RAG用途利用（R-4b）

### F-MTG05 会議I/O

- 【目的】会議の入出力を成立させる
- 【入力】音声・チャット
- 【出力】音声入出力・チャットログ・会議ログ
- 【成功条件】音声入出力とログ記録が動作する
- 【禁止事項】無関係データの送信

## 動画機能（PHASE11・2D動画のみ）

### F-VID01 待機表示

- 【目的】既定のアイドル表示を出す
- 【入力】無操作・通常状態
- 【出力】`待機` 動画
- 【成功条件】既定状態で `待機` が表示される
- 【禁止事項】3D/AR表示（R-12）

### F-VID02 話す表示

- 【目的】発話中の表示を出す
- 【入力】音声出力の開始/終了
- 【出力】`話す`（音声出力中ループ）→終了で `待機` へクロスフェード
- 【成功条件】音声出力開始と同時に再生・出力中ループ・終了でクロスフェード復帰
- 【禁止事項】黒画面 / ハードカット

### F-VID03 アイドリング表示

- 【目的】放置時の演出を出す
- 【入力】5秒無操作
- 【出力】`アイドリング1〜4` のいずれかをランダム→`待機` へクロスフェード
- 【成功条件】5秒無操作で1〜4のいずれかを再生し、`待機` へクロスフェード復帰
- 【禁止事項】黒画面 / ハードカット

### F-VID04 遷移ルール

- 【目的】黒画面のない遷移を保証する
- 【入力】全動画遷移
- 【出力】フェードイン/アウトのクロスフェード
- 【成功条件】全遷移で黒画面が発生しない
- 【禁止事項】黒画面 / ハードカット

## 外出モード（PHASE12）

### F-OUT01 外出シーケンス

- 【目的】外出演出を再生する
- 【入力】「外出」コマンド
- 【出力】動画1→6 を順再生→最後に2D入力画面で一時停止
  
  ```
  1. 外出
  2. ドア開ける
  3. 発進
  4. 後部座席
  5. 停車
  6. 研究室
  ```
- 【成功条件】1→6を順再生後、2D入力画面で停止
- 【禁止事項】3Dシーン化（入力画面は2D UIのみ・R-12）

### F-OUT02 ホログラム検索ループ

- 【目的】Obsidianをローカル検索する
- 【入力】検索ワード（音声 or テキスト）
- 【出力】Obsidian内検索結果の日本語報告＋再検索確認
- 【成功条件】Obsidianローカルのみを検索し、結果報告後に再検索を確認
- 【禁止事項】Web検索（R-7）

### F-OUT03 帰宅分岐

- 【目的】再検索/帰宅を分岐する
- 【入力】「もう一度」/「もういいよ」
- 【出力】「もう一度」→入力画面再表示／「もういいよ」→動画7→12を順再生
  
  ```
  7. 施錠
  8. ドア開ける
  9. 発進
  10. 後部座席
  11. 停車
  12. 帰宅
  ```
- 【成功条件】指定語に応じて再検索または7→12再生
- 【禁止事項】指定外動作

### F-OUT04 外出モードの遷移

- 【目的】黒画面のない遷移を保証する
- 【入力】1→6 / 入力画面前後 / 7→12 の全遷移
- 【出力】クロスフェード
- 【成功条件】全遷移で黒画面が発生しない
- 【禁止事項】黒画面 / ハードカット

```
待機
話す
アイドリング1 / アイドリング2 / アイドリング3 / アイドリング4
外出 / ドア開ける / 発進 / 後部座席 / 停車 / 研究室   （外出 1→6）
施錠 / ドア開ける / 発進 / 後部座席 / 停車 / 帰宅     （帰宅 7→12）
```

- 補足：「ドア開ける/発進/後部座席/停車」は外出・帰宅で重複。共有ファイルにするか `_外出`/`_帰宅` で分割するかは実装時にユーザー確認。
